# 项目说明

本项目是一个纯静态的幼儿识字练习应用。

## 命令行

- 执行 shell 命令时统一加 `rtk` 前缀。
- 文本搜索优先用 `rg`。
- 需要复杂文件谓词时用原生 `find`，不要依赖 `rtk find` 不支持的参数。

## 技术栈

- Vite + React + TypeScript。
- 项目已接入 Tailwind CSS，但当前主要样式集中在 `src/styles/global.css`。
- 图标使用 `lucide-react`。
- 开关控件使用 Radix UI。
- 拼音使用 `pinyin-pro` 生成。除非有明确原因，不要重新引入手工维护的拼音表。
- 图片识别使用 `tesseract.js` 在浏览器内完成。除非用户明确要求，不要添加服务端 OCR、账号系统或图片上传服务。
- 手写字形使用 `lxgw-wenkai-screen-webfont` 自托管静态字体资源，不依赖外部 CDN。
- 产物必须保持为纯静态 HTML/CSS/JS。不要添加后端、账号系统或服务端存储。
- 远程读音当前使用免费的有道 `dictvoice` 公开接口作为优先远程 TTS；它只能读文本，不支持通过拼音指定多音字读音。

## 核心产品规则

- 应用以移动端优先设计，必须兼容竖屏和横屏。
- 移动端布局使用 `100dvh`，并注意安全区间距。
- 不要依赖 hover 才能完成关键操作。
- 触控目标要足够大，适合手机和平板点击。
- 卡片模式的三个判断动作是：
  - `错误`
  - `巩固`
  - `正确`
- 三个判断按钮必须保持在同一行。
- 如果当前卡片还没有判断结果，点击判断按钮后自动进入下一张。
- 如果当前卡片已经有判断结果，再次点击判断按钮只修改结果，并停留在当前卡片。
- `上一个` 和 `下一个` 只负责导航，不能改变已保存的判断结果。
- 回到上一张卡片时，必须显示该卡片当前已选结果，并允许修改。
- 结果统计必须基于每个字的最新保存结果计算，不能基于历史点击记录叠加计算。
- 多音字必须允许家长在录入页选择本次练习读音。
- 多音字选中的拼音必须进入练习项、历史字表和分享链接；练习卡片显示选中的一个读音，不要在卡片上展示多个候选读音。
- 不需要兼容旧版 `chars=` 分享链接或旧版 `character-practice:v1` 本地历史数据，后续维护按当前新结构处理。
- 语音播放可以接收当前字的选中拼音，但有道远程 TTS 不能真正按拼音指定多音字读音；不要假设语音一定和注音一致。
- 支持方向键在按钮之间按空间位置移动焦点，并支持确定键触发当前聚焦按钮；不要破坏移动端触控。
- 上传图片识别字表必须支持多张图片批量识别，识别结果只提取汉字，按出现顺序去重后合并到当前输入字表。
- 图片识别必须在浏览器本地处理图片；中文 OCR 训练数据可以按需从 Tesseract.js 默认 CDN 下载并由浏览器缓存，但不要把用户图片发到后端。

## UI 约定

- 卡片注音显示在汉字上方。
- 注音和汉字的大小比例要协调，注音不能像很小的脚注。
- 汉字展示必须使用 `--font-character` 字体变量，支持标准、楷体、手写体切换；不要把练习字硬编码为系统黑体。
- 认字卡片的大字区域使用田字格，格线要辅助识字，不要压过汉字主体。
- 录入页的多音字候选读音使用按钮选择，触控目标要足够大，选中态要清楚且不能造成布局跳动。
- 判断按钮颜色：
  - `错误`：红色 / 珊瑚色
  - `巩固`：暖黄色
  - `正确`：绿色
- 判断按钮的选中态要稳定，不要让按钮上下位移或造成布局跳动。
- 导航和小控制优先使用常见图标按钮。
- 卡片只用于真正需要框起来的内容或面板，不要在卡片里嵌套卡片。
- 遥控器/键盘焦点态必须清晰可见，方向键导航优先按屏幕空间位置找最近可用按钮。
- 确定键应支持 `Enter`、`NumpadEnter`、`OK`、`Select` 和 Android `DPAD_CENTER` 常见键值。

## 本地存储

- 本地持久化通过 `src/storage/localStorage.ts` 使用 `localStorage`。
- 最近字表只保存在当前浏览器本地。
- 当前本地存储 key 为 `character-practice:v2`。
- 设置项包含 `characterFont`，默认值为 `handwriting`。
- 最近字表条目保存为 `CharacterDraft[][]`，单个字条目结构为 `{ char, pinyin? }`，不要退回纯 `string[][]`。
- 最近字表支持：
  - 使用
  - 编辑
  - 删除
  - 分享
- 编辑最近字表时，应把该字表加载到输入框。开始练习后，用修改后的字表替换原历史项，不要额外保留过期的旧记录。

## 分享

- 分享默认使用 `s=` 参数保存短链接编码，方便在微信中直接点击打开。
- 同时必须支持 `w=` 参数保存可读字表编码，方便用户手动直接用汉字写链接。
- `s=` 短链接编码按每个字 4 字节保存：3 字节 Unicode code point + 1 字节读音索引，然后转为 base64url。
- `w=` 可读格式中，普通字直接写汉字；多音字选择默认第一个读音时不标注；选择非默认读音时在汉字后直接追加一基读音序号，不加分隔符。序号 `1` 表示默认读音并省略，第二个读音从 `2` 开始，例如 `行2长2乐2`。
- 分享内容必须能恢复家长选择的多音字拼音；当前通过 `pinyin-pro` 的候选读音顺序和读音索引恢复。
- 不要新增服务端分享存储；分享仍然是纯 URL 参数。

## 语音

- `src/speech/speechSynthesis.ts` 负责读音播放，优先远程音频，失败后再降级 Web Speech。
- 免费公开远程 TTS 当前只使用有道 `https://dict.youdao.com/dictvoice`。
- 有道接口读汉字可返回音频，但实测不支持读 `háng`、`hang` 这类拼音输入；因此它不能解决多音字精确发音。
- 如需精确支持多音字发音，优先考虑纯静态音频资源或明确不暴露密钥的代理方案；不要把云厂商 API Key 放到前端代码或 GitHub Pages 环境中。

## 目录职责

- `src/core/`：纯练习逻辑和统计逻辑，保持独立于 React 和 DOM。
- `src/modes/`：练习模式 UI 和模式内逻辑。
- `src/pages/`：录入页、练习页、结果页等页面级组合。
- `src/ui/`：可复用 UI 基础组件。
- `src/storage/`：浏览器存储和存储类型。
- `src/speech/`：语音合成和音效。
- `src/ocr/`：图片 OCR 识别流程，保持独立于 React 页面组合。
- `src/styles/global.css`：当前视觉系统和响应式布局。
- `src/types/`：共享 TypeScript 类型。
- `src/utils/`：小型通用工具函数，包括遥控器方向键焦点导航。

## 验证

交付有意义的改动前，运行：

```bash
rtk npm run build
rtk npm audit
```

涉及 UI 改动时，用 in-app browser 验证：

- 手机竖屏，约 `390x844`
- 手机横屏，约 `844x390`

重点检查：

- 没有横向溢出
- 练习页没有不合理的纵向溢出
- 三个判断按钮始终保持同一行
- 选中态清晰可见，且不会造成布局跳动
- 卡片导航能保留并正确修改上一张的判断结果
- 方向键能按空间位置移动到邻近按钮，确定键能触发当前聚焦按钮
