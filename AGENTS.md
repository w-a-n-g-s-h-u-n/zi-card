# 项目协作说明

本文件只保留后续代理必须遵守的工程约束、产品状态模型和容易踩坑的实现边界。功能介绍以 README 和代码为准；维护本文件时优先重排、合并或删除过期内容，避免在末尾补丁式追加。

## 项目边界

- 这是一个纯静态的幼儿识字练习应用，技术栈为 Vite + React + TypeScript。
- 构建产物必须保持为 HTML/CSS/JS；不要添加后端、账号系统、服务端存储，或任何需要在前端暴露密钥的云服务。
- 移动端优先，必须兼容手机竖屏、手机横屏和平板触控；关键操作不能依赖 hover。
- 如果 README、旧注释或旧本地数据格式与本文件冲突，以当前代码和本文件的维护约束为准。

## 命令与验证

- 执行 shell 命令统一加 `rtk` 前缀。
- 文本搜索优先用 `rtk rg`。
- 需要复杂 `find` 谓词或 `-exec` 时使用 `rtk proxy find ...`，不要使用不支持这些参数的 `rtk find`。
- 代码或依赖改动交付前运行：

```bash
rtk npm run build
rtk npm audit
```

- 涉及 UI 改动时，用 in-app browser 验证约 `390x844` 竖屏和 `844x390` 横屏，重点看横向溢出、练习页纵向溢出、判断按钮同排、进度点居中、选中态不跳动、上一张结果可回显修改、方向键与确定键可用。
- UI 验证不要自行启动本地服务；使用用户已经启动好的本地服务和当前 in-app browser 页面。若服务不可访问，先告知用户并等待其启动或确认，不要主动运行 `rtk npm run dev`。

## 目录职责

- `src/core/`：纯练习逻辑、统计逻辑、分享编码解析；保持独立于 React 和 DOM。
- `src/pages/`：录入页、练习页、结果页等页面级组合。
- `src/pages/setup/`：录入页拆出的输入、OCR、设置、最近字表等子组件。
- `src/modes/`：练习模式 UI 和模式内逻辑。
- `src/storage/`：浏览器本地持久化和存储类型。
- `src/ocr/`：图片 OCR 流程；保持独立于 React 页面组合。
- `src/speech/`：语音合成和音效。
- `src/ui/`：可复用 UI 基础组件。
- `src/utils/`：通用工具函数，包括遥控器/方向键焦点导航。
- `src/styles/global.css`：当前主要视觉系统和响应式布局；Tailwind 已接入但不是主要样式来源。

## 依赖边界

- 图标使用 `lucide-react`；小控制和导航优先使用常见图标按钮。
- 开关控件使用 Radix UI。
- 小字卡片拖动排序使用 `@dnd-kit`，保持触控友好的排序动画；不要改回原生 HTML 拖放或鼠标优先方案。
- 拼音使用 `pinyin-pro` 生成；除非有明确理由，不要重新引入手工维护的拼音表。
- 手写字形使用 `lxgw-wenkai-screen-webfont` 自托管静态字体资源；字体 CSS 通过 `src/utils/handwritingFont.ts` 按需异步加载，不进入首屏全局 CSS。
- 图片识别使用 `@paddleocr/paddleocr-js`，不要重新引入 Tesseract 兼容逻辑。

## 字表与读音模型

- 最近字表保存在当前浏览器本地，单个字条目为 `CharacterDraft`：`{ char, pinyin? }`；不要退回纯 `string[][]`。
- 多音字必须允许家长在录入页选择本次练习读音；选中的拼音必须进入练习项、历史字表和分享链接。
- 练习卡片只显示选中的一个读音，不展示多个候选读音。
- 通过分享链接进入的字表默认不展示多音字候选按钮，但必须恢复链接中的已选读音。
- 只有用户修改字表内容、点击历史字表的编辑按钮、确认 OCR 候选后，才展示候选读音按钮；单纯聚焦输入框不算编辑。
- 录入页小字卡片支持直接拖动排序，不额外放拖动按钮或手柄。
- 开启编辑拼音时，小字卡片不能拖动，避免触控排序和读音选择冲突。

## 练习与结果规则

- 卡片模式只有三个判断动作：`错误`、`巩固`、`正确`，三个按钮必须保持同一行。
- 当前卡片还没有判断结果时，点击判断按钮后自动进入下一张；已有判断结果时，再次点击只修改结果并停留在当前卡片。
- `上一个` 和 `下一个` 只负责导航，不能改变已保存判断结果。
- 回到上一张卡片时，必须显示该卡片当前已选结果，并允许修改。
- 结果统计基于每个字的最新保存结果计算，不能基于历史点击记录叠加。
- 识字结果自动保存到本地历史，并按原始字表身份关联；同一字表的历史可合并展示，不同字表不要混在一起。
- 结果详情页必须支持查看原始字表、本轮练习字、错误、巩固、正确、未练分组，并提供“分享识字结果”“练待巩固字”“练原字表”。
- “练待巩固字”只练错误和巩固字，不包含未练字。

## 本地存储

- 本地持久化通过 `src/storage/localStorage.ts` 使用 `localStorage`。
- 当前本地存储 key 为 `character-practice:v2`；无需兼容旧版 `character-practice:v1`。
- 设置项包含 `characterFont`，默认值为 `handwriting`。
- 最近字表支持使用、编辑、删除、分享。
- 编辑最近字表时，应把该字表加载到输入框；开始练习后，用修改后的字表替换原历史项，不额外保留过期旧记录。
- `StoredData` 包含 `resultHistoriesByListIdentity: Record<string, PracticeResultRecord[]>`，每个字表最多保留最近 20 条结果。
- `PracticeResultRecord` 至少保留 `id/sessionId`、`sourceListIdentity`、`sourceDrafts`、`practiceDrafts`、`mode`、`round`、`createdAt/updatedAt` 和 `results`。
- 删除最近字表时必须同步删除该字表身份下的结果历史；删除单条结果历史不能影响字表本身。
- 导入分享结果时，如果本地没有同一字表，应把分享里的原始字表加入最近字表并挂上历史；如果已有同一字表，应合并到本地历史。
- 导入分享结果或恢复历史时，要按字表内容、已选拼音和练习结果做语义去重；不要只依赖记录 `id`。

## 分享协议

- 所有分享按钮都只复制分享链接并更新地址栏；不要调用 `navigator.share`，也不要针对微信或系统分享做特殊分支。
- 普通字表分享只使用 `s=` 参数；不要再识别或生成 `w=` 可读字表参数，也无需兼容旧版 `chars=`。
- `s=` 按每个字 4 字节保存：3 字节 Unicode code point + 1 字节读音索引，然后转为 base64url。
- 分享内容必须能恢复家长选择的多音字拼音；当前通过 `pinyin-pro` 的候选读音顺序和读音索引恢复。
- 识字结果分享使用 `s=` 加可选 `p=`、`e=`、`r=`、`u=`：`s` 保存原始字表，`p` 保存本轮练习字在原始字表里的下标，`e/r/u` 分别保存错误、巩固、未练下标。
- 存在 `p=` 时，`e/r/u` 下标基于 `p` 还原后的本轮练习字；没有 `p=` 时，`e/r/u` 下标基于完整原始字表。
- `p/e/r/u` 按索引数组编码为二进制字节后转 base64url；不要使用点分 base36、JSON payload、时间戳、版本号、mode 或 round。
- 空索引列表通常省略；完整原始字表全对且没有 `p/r/u` 时，用空 `e=` 作为“这是结果分享”的标记。
- 打开合法结果分享链接后要自动导入本地历史并进入结果详情；非法结果参数或旧格式参数只按普通 `s=` 字表分享处理。
- 不要新增服务端分享存储；分享仍然是纯 URL 参数。

## OCR

- OCR 初始化固定使用 WASM 后端。
- 首页不主动初始化 OCR；用户点击、触碰或聚焦 `识别图片` 前才预热模型。
- 移动端优先使用 PaddleOCR.js 原生 tar 加载，依赖同源静态站点/CDN 和浏览器 HTTP 缓存，不在手机浏览器里动态打包模型。
- 当前默认模型为 `PP-OCRv6_tiny_det` + `PP-OCRv6_tiny_rec`，模型 tar 放在 `public/models/paddleocr/` 并随静态站点同源分发。
- 上传图片识别字表必须支持多张图片批量识别；识别结果只提取汉字，按图片选择顺序去重后进入可编辑候选字表。
- 图片识别相关详情区域默认隐藏，只展示上传按钮；上传图片后才展开识别状态、原图预览、识别结果和候选字表。
- 识别结果必须展示原图预览供用户对比，并由用户确认后才合并到当前输入字表，继续复用现有多音字选择流程。
- 图片必须在浏览器本地处理；不要把用户图片发到后端，也不要接入需要在前端暴露密钥的云 OCR。

## 语音与键盘/遥控器

- `src/speech/speechSynthesis.ts` 负责读音播放，优先远程音频，失败后再降级 Web Speech。
- 免费公开远程 TTS 当前只使用有道 `https://dict.youdao.com/dictvoice`。
- 语音播放可以接收当前字的选中拼音，但有道远程 TTS 只能读文本，不能按拼音指定多音字读音；不要假设语音一定和注音一致。
- 如需精确支持多音字发音，优先考虑纯静态音频资源或明确不暴露密钥的代理方案。
- 支持方向键在按钮之间按屏幕空间位置移动焦点，并支持确定键触发当前聚焦按钮；不要破坏移动端触控。
- 确定键应支持 `Enter`、`NumpadEnter`、`OK`、`Select` 和 Android `DPAD_CENTER` 常见键值。

## UI 约定

- 移动端布局使用 `100dvh`，并注意安全区间距。
- 触控目标要足够大，适合手机和平板点击。
- 卡片注音显示在汉字上方，大小比例要协调，不能像很小的脚注。
- 汉字展示必须使用 `--font-character` 字体变量，支持标准、楷体、手写体切换；不要把练习字硬编码为系统黑体。
- 认字卡片的大字区域使用田字格，格线要辅助识字，不要压过汉字主体。
- 录入页多音字候选读音使用按钮选择，触控目标要足够大，选中态清楚且不能造成布局跳动。
- 判断按钮颜色：`错误` 用红色/珊瑚色，`巩固` 用暖黄色，`正确` 用绿色。
- 判断按钮选中态要稳定，不要让按钮上下位移或造成布局跳动。
- 练习页进度点表现为横向滑动轨道，当前点必须始终保持在进度条容器中间；不要让当前点放大、列表长度或当前索引变化造成中心漂移。
- 卡片只用于真正需要框起来的内容或面板，不要在卡片里嵌套卡片。
- 遥控器/键盘焦点态必须清晰可见。
