# 识字小练习

纯静态的幼儿认字练习网页。每次打开后录入本次字表，在浏览器内完成练习、复习和统计，不依赖后端服务。

## 功能

- 每次使用时录入本次要认的汉字，自动去重并生成注音。
- 支持多音字读音选择，练习卡片只显示本次选中的读音。
- 支持卡片模式和找字模式。
- 卡片模式提供 `错误`、`巩固`、`正确` 三种判断。
- 判断结果会显示在进度点中，并在结果页统计。
- 最近字表保存在当前浏览器本地，支持使用、编辑、删除和分享。
- 分享链接使用可读的 `w=` 参数，能携带多音字读音选择。
- 移动端优先，兼容手机竖屏、横屏和平板使用。
- 支持方向键在按钮间移动焦点，并支持确定键触发当前按钮，适合遥控器基础操作。

## 使用

1. 在首页输入本次要练习的字。
2. 如果出现多音字，在字表预览里选择本次读音。
3. 按 `开始` 进入练习。
4. 在卡片模式中选择：
   - `错误`
   - `巩固`
   - `正确`
5. 回到上一张卡片时，可以看到并修改之前的判断结果。

## 分享

点击 `分享` 会把当前字表写入 URL，并尽量调起系统分享；不支持系统分享时会复制链接。

分享参数示例：

```txt
?w=行2长2乐2
```

分享内容完全保存在 URL 中，不依赖服务器。普通字直接写汉字；如果多音字选择的是默认第一个读音，就不额外标注；选择其他读音时，在汉字后直接追加从 `2` 开始的读音序号。

## 语音

读音优先使用免费的有道 `dictvoice` 远程音频，失败后降级为浏览器 Web Speech。

需要注意：有道公开接口只能读文本，不能指定多音字读音。因此多音字卡片的注音会按选择显示，但语音不保证一定读成选中的多音字读音。

## 开发

```bash
npm install
npm run dev
```

## 构建

```bash
npm run build
```

本项目是纯静态产物，构建后输出到 `dist/`。

## GitHub Pages

项目已支持 GitHub Pages。

### 自动部署

仓库推送到 `main` 或 `master` 后，`.github/workflows/deploy-pages.yml` 会自动：

1. 安装依赖
2. 构建静态文件
3. 上传 `dist`
4. 发布到 GitHub Pages

首次使用时，在 GitHub 仓库中打开：

```txt
Settings -> Pages -> Build and deployment -> Source -> GitHub Actions
```

然后推送代码即可。

### 路径说明

`vite.config.ts` 使用 `base: "./"`，因此同一份构建产物可以部署在：

- `https://用户名.github.io/仓库名/`
- 自定义域名根路径

无需根据仓库名手动修改 `base`。
