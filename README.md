# 识字小练习

纯静态的幼儿认字练习网页。每次打开后录入本次字表，在浏览器内完成练习、复习和统计，不依赖后端服务。

## 功能

- 每次使用时录入本次要认的汉字，自动去重并生成注音。
- 支持多音字读音选择；进入字表编辑后，可在字表预览标题栏直接打开读音编辑，练习卡片只显示本次选中的读音。
- 字表预览、练习设置和结果分组里的小字卡支持拖动调整顺序；结果页和详情页的调整只影响本轮结果顺序，不会改原始字表。
- OCR 模型加载成功后才显示图片识别按钮；支持上传一张或多张图片，在浏览器内识别并生成可编辑候选字表。
- 支持单字大卡和多字平铺两种识字展示方式。
- 单字大卡提供 `错误`、`巩固`、`正确` 三种判断；多字平铺可以点击每个字循环标记状态。
- 拼音显示、字卡进度、音效、练习展示方式和汉字字形可以在设置里调整，并在首页和练习中保持一致。
- 汉字字形可切换为标准、楷体或手写体；楷体和手写体使用内置静态字体资源，并按需异步加载。
- 认字卡片使用田字格辅助观察字形。
- 判断结果会显示在进度点中，并在结果页统计。
- 最近字表和识字结果保存在当前浏览器本地，支持使用、取消选择、编辑、删除、查看历史和分享。
- 结果详情支持查看原始字表、本轮练习字、错误、巩固、正确和未练分组。
- 支持分享字表和识字结果；分享只复制链接并更新地址栏，不会调起系统分享。
- 移动端优先，兼容手机竖屏、横屏和平板使用。
- 支持方向键在按钮间移动焦点，并支持确定键触发当前按钮，适合遥控器基础操作。

## 使用

1. 在首页输入本次要练习的字；输入框为空时不会显示字表预览。
2. 也可以在 `识别图片` 按钮出现后上传图片，识别出的汉字会进入候选字表。
3. 可以点击最近字表整块区域选中字表；再次点击同一个字表会取消选中并清空输入。
4. 选中最近字表后默认只显示字表预览。需要改字表时，在齿轮设置菜单中点 `编辑字表`，编辑完成后点 `退出编辑` 回到预览。
5. 如果出现多音字，进入字表编辑后点预览标题栏的 `编辑读音`，在字表预览里选择本次读音；没有多音字时不会显示这个入口。
6. 检查字表预览，必要时拖动小字卡调整顺序，再按 `开始` 进入练习。
7. 在单字大卡中选择：
   - `错误`
   - `巩固`
   - `正确`
8. 回到上一张卡片时，可以看到并修改之前的判断结果。
9. 如果切换到多字平铺，可以直接点击每个字循环 `未判 -> 错误 -> 巩固 -> 正确 -> 未判`，点 `保存` 后进入结果。
10. 完成本轮后可以查看结果详情，分享识字结果，或只练错误和巩固字。

## 本地历史

最近字表只保存在当前浏览器本地。每个字表支持使用、取消选择、编辑、删除、查看历史和分享；点击最近字表的整块区域即可使用该字表，再次点击同一个已选中字表会取消选择。选中字表后默认只显示预览，不显示输入框；需要修改时，通过预览区齿轮菜单进入 `编辑字表`。

编辑最近字表时，顺序或拼音改动会立即更新本地缓存；开始练习后，会用修改后的字表替换原来的本地记录。读音编辑和字表文本编辑分开：读音入口显示在字表预览标题栏，只在有多音字且当前字表可编辑时出现。

识字结果会自动保存到本地历史，并按原始字表关联。同一字表的多次练习会合并展示，不同字表不会混在一起。删除最近字表时，会同步删除该字表下的结果历史；删除某一条结果历史不会影响字表本身。

结果详情页会展示原始字表、本轮练习字、错误、巩固、正确和未练分组，并提供：

- `分享识字结果`
- `练待巩固字`
- `练原字表`

其中 `练待巩固字` 只包含错误和巩固字，不包含未练字。

## 图片识别

图片识别使用 `@paddleocr/paddleocr-js` 在浏览器本地完成，图片不会上传到服务器，也不需要云 OCR 密钥。当前模型为 `PP-OCRv6_tiny_det` 和 `PP-OCRv6_tiny_rec`，模型 tar 放在 `public/models/paddleocr/`，随静态站点同源分发。

识别前会在本地做图片方向修正、白底、长边限制、灰度和轻微对比度增强。识别结果只提取汉字，按图片选择顺序合并并去重，然后进入可编辑候选字表；用户确认时会同时看到原图缩略图用于对比，确认后才会合并进当前字表。

为兼顾首页速度和移动端稳定性，首页会静默预热 OCR，只有模型加载成功后才显示 `识别图片` 按钮；加载失败不会弹通知。点击、触碰或聚焦图片识别入口时也会尝试预热。部署到 GitHub Pages + Cloudflare 后，两个 V6 tiny tar 会由同源静态站点和 CDN 分发，并依赖浏览器 HTTP 缓存复用。

## 分享

点击分享按钮会把分享内容写入 URL、更新地址栏，并复制链接。应用不会调用系统分享，也不会针对微信或其他平台做特殊分支。

普通字表分享只使用 `s=` 参数：

```txt
?s=AIhMAQCVfwEATlAB
```

`s=` 中保存了字表里的汉字和已选择的多音字读音索引。分享内容完全保存在 URL 中，不依赖服务器。

通过普通字表分享链接打开时，会先恢复已选读音，但不会立即展开多音字候选按钮；用户修改字表内容、点击历史字表的编辑按钮或确认 OCR 候选后，才会显示读音选择。

打开分享链接时，如果本地已经有同一组汉字，会先对比本地和分享内容。顺序不同会进入顺序对比页面，一次性选择使用本地顺序或分享顺序；拼音不同会进入单独的拼音对比页面，只展示读音不同的字，并且每个字都可以单独选择使用本地读音或分享读音。顺序和拼音分开处理，选择顺序不会覆盖已经选好的拼音。

识字结果分享在 `s=` 的基础上增加可选的 `p=`、`e=`、`r=`、`u=` 参数。其他用户打开合法结果分享链接后，会自动导入本地历史并进入结果详情；如果本地顺序和分享顺序不同，应用会按汉字重新映射结果索引，保证错误、巩固、正确和未练不会因为顺序选择而错位。如果结果参数不合法，则只按普通 `s=` 字表分享处理。

参数规则如下：

- `s=`：必填，保存原始字表。每个字使用 4 字节编码：3 字节 Unicode code point + 1 字节读音索引，再转成 base64url。
- `p=`：可选，保存本轮练习字在原始字表里的下标。没有 `p=` 时，表示本轮练习字就是完整原始字表。
- `e=`：可选，保存错误字下标。
- `r=`：可选，保存巩固字下标。
- `u=`：可选，保存未练字下标。

下标基准取决于 `p=`：有 `p=` 时，`e/r/u` 的下标基于 `p` 还原后的本轮练习字；没有 `p=` 时，`e/r/u` 的下标基于完整原始字表。索引不能重复或越界；`e/r/u` 三组之间不能重叠。`p/e/r/u` 都使用索引数组的二进制压缩后再转 base64url，不使用 JSON、时间戳、版本号、模式或轮次。

空索引列表通常省略。一个特殊情况是：完整原始字表全对，且没有 `p/r/u` 时，会保留空的 `e=`，用来标记“这是识字结果分享”，避免和普通字表分享混淆。

## 语音

读音优先使用免费的有道 `dictvoice` 远程音频，失败后降级为浏览器 Web Speech。

需要注意：有道公开接口只能读文本，不能指定多音字读音。因此多音字卡片的注音会按选择显示，但语音不保证一定读成选中的多音字读音。

```bash
npm install

```

## 构建

```bash
npm run build
```

本项目是纯静态产物，构建后输出到 `dist/`。

## GitHub Pages

项目已支持 GitHub Pages。

### 自动部署

仓库推送到 `main` 或 `master` 后，`.github/workflows/deploy-pages.yml` 会自动：

1. 安装依赖
2. 构建静态文件
3. 上传 `dist`
4. 发布到 GitHub Pages

首次使用时，在 GitHub 仓库中打开：

```txt
Settings -> Pages -> Build and deployment -> Source -> GitHub Actions
```

然后推送代码即可。

### 路径说明

`vite.config.ts` 使用 `base: "./"`，因此同一份构建产物可以部署在：

- `https://用户名.github.io/仓库名/`
- 自定义域名根路径

无需根据仓库名手动修改 `base`。
